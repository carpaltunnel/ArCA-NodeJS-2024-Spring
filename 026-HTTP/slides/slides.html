HTTP





Hyper Text Transfer Protocol

The Basics

HTTP (Hyper Text Transfer Protocol) is a client-server protocol that is used to fetch resources over the internet. Requests are made by a client, to a server, and the server responds accordingly. The most ubiquitous example is a web page

  1. Client (browser) requests a web page from a server.
  2. Server accepts the request and transfers the page data (the response) to the client
  3. Client loads the page data and displays it to the user

Stateless

HTTP is inherently a stateless protocol in that the server simply responds to requests from the client without storing state information about the client. Each request can be made in isolation and is not dependent on previous interactions.


That said, we've built a lot of stateful things on top of HTTP, like cookies and sessions, but it is truly stateless at the core.

TCP (Transmission Control Protocol)

HTTP is built on top of TCP. We won't talk much about TCP since it lives at a different layer and is abstracted away for us, but you should know it's there. The general flow of a HTTP request, with a nod to TCP, is as follows :

  1. Client opens a TCP connection to the server
  2. Client sends an HTTP request over the TCP connection to the server
  3. Server responds to the request with an HTTP response.
  4. Client consumes the response
  5. Client closes or reuses the TCP connection.

HTTP Requests

HTTP requests have the following components that we will shortly discuss :

  • An HTTP Method
  • The URL for the requested resource
  • The HTTP protocol version
  • HTTP headers
  • Optional data body

HTTP Responses

HTTP responses have the following components :

  • The HTTP protocol version
  • A status code
  • A status message
  • HTTP Headers
  • Optional data body

HTTP Methods (verbs)

HTTP request methods indicate the action the request is expected to perform. For this reason, they are often called "verbs" since they dictate an action to take. We'll just discuss the methods used in a RESTful API and ignore the others for now.

  • GET - Retrieve data
  • POST - Submit data, usually to create a new entry
  • PUT - Update data, replace an existing entry with new data
  • DELETE - Delete data
  • PATCH - Update data, but only a partial modification rather than the full replace of a PUT

HTTP Status Codes

HTTP Response status codes are simple numbers that indicate the state of the response.

  1. The Good : 200s
  2. The Wandering : 300s
  3. The Bad : 400s
  4. The Ugly : 500s

HTTP Headers

Headers are extra informational fields that are added to HTTP communications - both from the client and server side.

  1. content-type
  2. content-length
  3. user-agent
  4. authorization
  5. x-request-id (and X- headers in general

Ports

While the TCP ports used for HTTP connections can be arbitrary, the default for HTTP is 80 and the default for HTTPS is 443. In your web service development you can choose pretty much any port (that isn't already in use) so long as you direct your requests to that port.


Network devices (load balancers, gateways, proxies, etc) and system port limitations complicate this a bit.
