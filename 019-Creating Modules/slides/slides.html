Modules






We've looked at creating classes in a single script but, as your projects grow more complex, you'll want to separate them out into self contained modules. Modules can basically be thought of as classes that expose some or all of their functionality.

Creating a Module

'use strict';
class Vehicle {
    constructor (make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
    }
    
    myMake() {
        console.log(`My make is : ${this.make}`);
    }
}
    
module.exports = Vehicle;

Using a Module

'use strict';
const Vehicle = require('./vehicle');

// Create instance of Vehicle and invoke constructor
const myVehicle = new Vehicle('Tesla', 'S', 2020);
myVehicle.myMake();

module.exports

There are three common ways to use module.exports :
  1. Export an entire class
  2. Export specific properties
  3. Export specific functions

Global Singletons

Global Singletons need to be used judiciously but they can be very helpful in certain scenarios.

'use strict';
class GlobalSingleton {
    static aFunction() {
    // just a function
    }
}

module.exports = new GlobalSingleton();
